# Specification: AST-to-Forth Emitter (Go-to-WASM)

## 1. Overview
This document defines the specification for the AST-to-Forth Emitter, which transpiles Go source code (via `go2json` AST) into Forth-based WASM instructions. The emitter enforces a strict architectural split between high-performance internal concurrency ("Fast Path") and system-level I/O ("Slow Path").

## 2. Architectural Split

### 2.1. The "Fast Path" (Internal Concurrency)
*   **Purpose:** Channel operations (`ch <- x`, `<-ch`), shared memory access, and goroutine scheduling.
*   **Implementation:** Managed entirely within the transpiler's runtime (browser memory space).
*   **Mechanism:** Uses `SharedArrayBuffer` and `Atomics` for shared memory access.
*   **Performance:** Nanoseconds. Bypasses the kernel and VFS.

### 2.2. The "Slow Path" (Wanix I/O)
*   **Purpose:** `fmt.Println`, file I/O, network requests.
*   **Implementation:** Managed by the Wanix kernel.
*   **Mechanism:** Standard WASI syscalls.
*   **Performance:** Milliseconds (acceptable for I/O).

## 3. Emitter Logic Specification

### 3.1. AST Traversal
The Emitter must recursively traverse the JSON AST generated by `go2json` (which follows the standard Go AST structure).

### 3.2. Mapping Rules (Go AST to Forth)

| Go AST Node | Description | Forth/WASM Mapping | Path |
| :--- | :--- | :--- | :--- |
| `FuncDecl` | Function declaration | `: name ... ;` | Fast |
| `BlockStmt` | Block of statements | Sequence of Forth words | Fast |
| `ExprStmt` | Expression statement | Forth word execution | Fast |
| `CallExpr` | Function call | Forth word call | Fast/Slow |
| `SelectorExpr` | `fmt.Println` | `HOST-LOG` (Slow) or Forth word (Fast) | Fast/Slow |
| `BasicLit` | String/Number literal | `S" ..."` or `n` | Fast |
| `IfStmt` | If statement | `IF ... ELSE ... THEN` | Fast |
| `ForStmt` | For loop | `BEGIN ... UNTIL` | Fast |
| `ReturnStmt` | Return statement | `EXIT` | Fast |

## 4. Memory Model Specification

### 4.1. Personal Memory (Private)
*   **Scope:** Isolated to each WebAssembly Instance (Goroutine).
*   **Size:** 64KB per instance.
*   **Contents:** Data Stack, Return Stack, local variables.

### 4.2. Shared Memory (The Heap)
*   **Scope:** Global, shared across all WebAssembly Instances.
*   **Size:** Configurable `SharedArrayBuffer`.
*   **Contents:** Global variables, large structs, Channels.
*   **Synchronization:** Atomic operations (`i32.atomic.wait`, `i32.atomic.notify`).
